# Kiến Trúc Dự Án

Tài liệu này mô tả kiến trúc của dự án **lifetek-frontend**, nêu bật cấu trúc, các module và trách nhiệm của chúng.

---

## Cấu Trúc Thư Mục

Dưới đây là cái nhìn tổng quan về các thư mục và tệp chính trong dự án:

---

```
lifetek-frontend/
├── public/ # Static assets like images, fonts, etc.
└── src/ # Source code
    ├── assets/ # Project assets (images, icons, etc.)
    ├── auth/ # Authentication-related logic
    ├── components/ # Reusable UI components
    ├── hooks/ # Custom React hooks
    ├── layouts/ # Layout components for structuring pages
    ├── lib/ # Utility functions and libraries
    ├── pages/ # Page components (route views)
    ├── routes/ # Routing definitions
    ├── sections/ # Page sections (composed UI components)
    ├── utils/ # Helper functions and utilities
├── .env # Environment variables
├── .gitlab-ci.yml # GitLab CI/CD pipeline configuration
├── tailwind.config.js # Tailwind CSS configuration
├── tsconfig.json # TypeScript configuration
└── vite.config.ts # Vite build tool configuration
```

## Các Module Chính

### **`app/`**

- Bao gồm trạng thái toàn cục, các provider và quản lý context cho ứng dụng.

### **`assets/`**

- Lưu trữ các tài nguyên đặc trưng của dự án, bao gồm hình ảnh và biểu tượng.

### **`auth/`**

- Chứa logic xác thực cho đăng nhập, đăng ký và quản lý phiên làm việc.

### **`components/`**

- Lưu trữ các thành phần UI có thể tái sử dụng như nút bấm, modal và các phần tử form.
- Khuyến khích tính nhất quán và tái sử dụng trong toàn bộ ứng dụng.

### **`hooks/`**

- Chứa các hook React tùy chỉnh để bao encapsulate các logic tái sử dụng, như gọi API hoặc quản lý trạng thái.

### **`layouts/`**

- Định nghĩa cấu trúc và layout của các trang, bao gồm header, footer và sidebar.

### **`lib/`**

- Lưu trữ các hàm tiện ích chung và các tích hợp của bên thứ ba.

### **`locales/`**

- Chứa các tệp ngôn ngữ phục vụ cho quốc tế hóa (i18n).

### **`pages/`**

- Đại diện cho các view chính của ứng dụng, mỗi view tương ứng với một route.

### **`routes/`**

- Quản lý logic routing cho ứng dụng, bao gồm các route động và tĩnh.

### **`sections/`**

- Các section cấp cao kết hợp các thành phần nhỏ hơn, được sử dụng để xây dựng các trang.

### **`utils/`**

- Các hàm tiện ích đa năng cho các tác vụ như định dạng, xác thực hoặc phản hồi API.

---

## Các Tệp Cấu Hình Cơ Bản

### **`.gitlab-ci.yml`**

- Tự động hóa các pipeline CI/CD cho việc kiểm thử, build và triển khai dự án.

### **`vite.config.ts`**

- Cấu hình Vite cho quá trình build, bao gồm các plugin và alias.

### **`tailwind.config.js`**

- Tùy chỉnh Tailwind CSS để đảm bảo phong cách đồng nhất trên toàn dự án.

### **`postcss.config.js`**

- Tích hợp các plugin PostCSS để xử lý CSS nâng cao.

---

## Ghi Chú Triển Khai

- Build sẵn sàng cho sản xuất được tạo trong thư mục `dist/` và có thể triển khai sử dụng các máy chủ web như **Nginx** hoặc **Apache**.
- Cấu hình riêng cho từng môi trường nên được quản lý thông qua tệp `.env`.

---

## Kết Luận

Kiến trúc này nhấn mạnh tính mô-đun, khả năng mở rộng và bảo trì, cung cấp một nền tảng vững chắc cho việc phát triển frontend. Các điều chỉnh hoặc cải tiến có thể dễ dàng được tích hợp mà không làm ảnh hưởng đến cấu trúc tổng thể.

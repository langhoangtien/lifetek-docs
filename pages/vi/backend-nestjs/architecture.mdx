### Cấu trúc dự án

Tài liệu này mô tả kiến trúc của dự án.

---

[[toc]]

### `.gitlab-ci.yml`

File này là file cấu hình cho các pipeline CI/CD của GitLab. Nó định nghĩa các giai đoạn, công việc, và các script cần thiết để tự động hóa các tác vụ như build, test, và triển khai ứng dụng.

---

## `.husky`

Husky là công cụ để ngăn chặn các commit hay push không hợp lệ, và nhiều hơn nữa. Xem [tài liệu Husky](https://typicode.github.io/husky/) để biết thêm chi tiết.

---

## `docs`

Thư mục lưu trữ tất cả các file tài liệu của dự án.

---

## `src`

Thư mục chứa tất cả các file mã nguồn của dự án.

---

### `api`

Nơi chứa các API của dự án.

---

### `background`

Thư mục dành cho các tác vụ nền.

---

### `common`

Thư mục chứa các file TypeScript dùng chung, ví dụ như DTOs, interfaces, types.

#### `dto`

Thư mục chứa tất cả các DTOs.

#### `interfaces`

Thư mục lưu trữ các [interface](https://www.typescriptlang.org/docs/handbook/interfaces.html) trong TypeScript.

#### `types`

Thư mục chứa các [type](https://www.typescriptlang.org/docs/handbook/2/everyday-types.html) trong TypeScript.

---

### `config`

Nơi lưu trữ các file cấu hình ứng dụng.

---

### `constants`

Thư mục lưu trữ các hằng số.

---

### `database`

Thư mục lưu trữ các file liên quan đến cơ sở dữ liệu.

#### `config`

Thư mục lưu trữ các file cấu hình cơ sở dữ liệu.

#### `decorators`

Thư mục lưu trữ các decorator liên quan đến cơ sở dữ liệu.

#### `entities`

Thư mục chứa các entity chung của ứng dụng.

#### `migrations`

Thư mục chứa các migration được tạo bởi TypeORM.

#### `seeds`

Thư mục chứa các seed để thêm dữ liệu cần thiết cho môi trường phát triển.

#### `data-source.ts`

File cấu hình data source cho các migration.

---

### `decorators`

Thư mục chứa các [decorator](https://www.typescriptlang.org/docs/handbook/decorators.html) toàn cục.

---

### `exceptions`

Thư mục chứa các class exception toàn cục.

---

### `filters`

Thư mục lưu trữ các [filter](https://docs.nestjs.com/exception-filters) ở mức ứng dụng.

---

### `generated`

Chứa các file TypeScript được sinh tự động, ví dụ `i18n.generated.ts` được tạo bởi [nestjs-i18n](https://www.npmjs.com/package/nestjs-i18n).

---

### `guards`

Thư mục lưu trữ các guard.

---

### `i18n`

Thư mục lưu trữ các file JSON dùng cho nội địa hóa (i18n).

---

### `interceptors`

Thư mục lưu trữ các [interceptor](https://docs.nestjs.com/interceptors).

---

### `libs`

Thư mục chứa các module dùng chung. Xem thêm tài liệu [NestJS modules](https://docs.nestjs.com/modules).

---

### `mail`

Bao gồm dịch vụ mailer, logic gửi email, sử dụng [nodemailer](https://nodemailer.com/about/) và handlebars để xử lý template.

---

### `shared`

Thư mục chứa các module dùng chung với các dịch vụ singleton toàn cục.

---

### `utils`

Thư mục lưu trữ các hàm tiện ích.

---

### `app.module.ts`

Module gốc của ứng dụng.

---

### `main.ts`

File entry chính của ứng dụng.

---

## `test`

Thư mục lưu trữ tất cả các file test e2e.

---

## `.env`

Các biến môi trường sẽ được tải trước khi ứng dụng khởi chạy và lưu trong `process.env`. (\*) là tên môi trường (local, development, test, staging, production).

---

Danh sách các file và thư mục mà bạn muốn bỏ qua trong repository git.

---

File cấu hình Docker Compose dành cho phát triển cục bộ.

## `lint-staged.config.mjs`

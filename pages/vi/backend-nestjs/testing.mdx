# Testing

Chúng tôi sử dụng **Jest** để thực hiện kiểm thử đơn vị (unit test) toàn diện trong dự án NestJS của mình. Dưới đây là các lệnh và phương pháp bạn có thể sử dụng để chạy kiểm thử hiệu quả.

---

[[toc]]

## Kiểm thử đơn vị (Unit Testing)

### Chạy một file kiểm thử cụ thể

Để chạy kiểm thử trong một file cụ thể, sử dụng lệnh sau:

```bash copy
pnpm test path/to/file

# ví dụ: pnpm test src/api/auth/auth.controller.spec.ts
```

## Kiểm thử End-to-End (E2E)

### Chạy kiểm thử End-to-End (e2e)

Kiểm thử End-to-End có thể được thực thi bằng lệnh:

```bash copy
pnpm test:e2e
```

## Tất cả kiểm thử

### Chạy tất cả các kiểm thử

Thực thi toàn bộ các trường hợp kiểm thử trong dự án bằng lệnh:

```bash copy
pnpm test
```

### Chạy kiểm thử ở chế độ Watch

Để kiểm thử liên tục, chế độ watch cho phép bạn tự động chạy lại kiểm thử khi có thay đổi trong file:

```bash copy
pnpm test:watch
```

### Chạy kiểm thử với báo cáo Coverage

Để tạo báo cáo độ phủ mã (code coverage), sử dụng lệnh:

```bash copy
pnpm test:cov
```

## Kiểm thử trên IDE

### Visual Studio Code

Để có trải nghiệm kiểm thử liền mạch trong Visual Studio Code, cài đặt tiện ích mở rộng [Jest Runner](https://marketplace.visualstudio.com/items?itemName=firsttris.vscode-jest-runner). Tiện ích này cho phép bạn chạy kiểm thử trực tiếp từ file kiểm thử, giúp đơn giản hóa quy trình phát triển.

![Jest Runner](https://github.com/firsttris/vscode-jest/raw/master/public/vscode-jest.gif)

Bằng cách làm theo hướng dẫn này, bạn có thể đảm bảo quy trình kiểm thử hiệu quả, dẫn đến mã nguồn đáng tin cậy và dễ bảo trì hơn.

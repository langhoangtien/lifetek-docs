## Các Endpoint

Chúng tôi sử dụng Swagger để tài liệu hóa các endpoint API. Bạn có thể truy cập tài liệu API bằng cách truy cập route `/api-docs` trên trình duyệt.

Ví dụ, nếu ứng dụng của bạn đang chạy trên `http://localhost:3000`, bạn có thể truy cập tài liệu API tại [http://localhost:3000/api-docs](http://localhost:3000/api-docs).
